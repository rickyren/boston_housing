编程练习3：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

或

  • (可选) 不使用任何外部库，参考决定系数的定义进行计算，这也可以帮助你更好的理解决定系数在什么情况下等于0或等于1。

def performance_metric(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    
    from sklearn.metrics import r2_score
    score = r2_score(y_true, y_predict)

    return score

def performance_metric2(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    
    n = len(y_true)
    assert len(y_predict) == n
    
    y_bar = np.mean(y_true)
    ss_tot = sum([(y_true[i] - y_bar) ** 2 for i in range(n)])
    ss_res = sum([(y_true[i] - y_predict[i]) ** 2 for i in range(n)])
    score = 1 - ss_res / ss_tot

    return score

问题 8 - 交叉验证¶

  • 什么是K折交叉验证法（k-fold cross-validation）？
  • GridSearchCV是如何结合交叉验证来完成对最佳参数组合的选择的？
  • GridSearchCV中的'cv_results_'属性能告诉我们什么？
  • 网格搜索时如果不使用交叉验证会有什么问题？交叉验证又是如何解决这个问题的？

提示： 在下面 fit_model函数最后加入 print pd.DataFrame(grid.cv_results_) 可以帮你查看更多信息。

问题 8 - 回答：¶

  • K折交叉验证是将训练数据集几乎平均分为$K$份(默认顺序切分，但也可以申明随机切分），并依次将其中一份作为交叉验证集，其他$K-1$份作为训练集，从而进行交叉验证，对每组训练、验证组合按照给定的评价函数计算评分，最终得出评分的均值和标准差，从而较为客观的评估当前模型的表现。
  • GridSearchCV在采用网格搜索尝试所有可能的参数取值时，通过在训练集上进行交叉验证来更加客观地评估模型表现，通过比较K折交叉验证所获得的评分的平均值，选出评分平均值最好的参数组合，作为最佳参数组合。
  • 'cv_results_'中包含了网格搜索所尝试的各种参数组合的实际交叉验证结果，包括分数均值、标准差、耗时等。虽然 GridSearchCV 已经提供了根据这些结果选定的最有模型，但这里的结果可以用来进一步检验最有模型的选择是否合理，以及是否需要微调，例如有些情况下，模型性能非常重要，可能需要结合耗时，舍弃一部分准确率。
  • 如果不使用交叉验证，而直接使用模型在训练集上的评分对模型的评估，可能受到过拟合的影响。交叉验证通过将一部分数据排除在训练之外，并用来验证模型的表现，可以发现过拟合的情况，从而更加客观地预测模型的表现。而如果简单地将训练集按一定比例进一步划分为训练和验证集来进行验证，就会进一步较少训练模型可用的数据，从而影响训练效果。K折交叉验证的优势在于，其既进行了充分的交叉验证，排除过拟合的情况，又将所有训练数据纳入模型训练当中，最大限度减轻因为训练数据减少对模型效果的影响。

def fit_model(X, y):
    """ 基于输入数据 [X,y]，利于网格搜索找到最优的决策树模型"""
    
    from sklearn.model_selection import KFold, GridSearchCV
    from sklearn.tree import DecisionTreeRegressor
    from sklearn.metrics import make_scorer
    
    cross_validator = KFold()
    
    regressor = DecisionTreeRegressor()

    params = {"max_depth": range(1, 11)}

    scoring_fnc = make_scorer(performance_metric)

    grid = GridSearchCV(regressor, params, scoring_fnc, cv = cross_validator)

    # 基于输入数据 [X,y]，进行网格搜索
    grid = grid.fit(X, y)

    # 返回网格搜索后的最优模型
    return grid.best_estimator_

def cross_val_score(estimator, X, y, scoring = performance_metric, cv=3):
    """ 返回每组交叉验证的模型分数的数组 """
    
    scores = np.zeros(cv)
    m = len(y)
    for i in range(cv):
        train_pos = [x for x in range(m) if x % cv != i]
        val_pos = [x for x in range(m) if x % cv == i]
        estimator.fit(X.iloc[train_pos], y.iloc[train_pos])
        scores[i] = scoring(y.iloc[val_pos], estimator.predict(X.iloc[val_pos]))
        
    return scores

def fit_model2(X, y):
    """ 基于输入数据 [X,y]，利于网格搜索找到最优的决策树模型"""
    from sklearn.tree import DecisionTreeRegressor
    
    best_score = -100
    for param in range(1,11):
        clf = DecisionTreeRegressor(max_depth = param)
        score = np.mean(cross_val_score(clf, X, y))
        if score > best_score:
            best_score = score
            best_estimator = clf
    
    #最优交叉验证分数对应的最优模型
    
    return best_estimator

模型健壮性¶

一个最优的模型不一定是一个健壮模型。有的时候模型会过于复杂或者过于简单，以致于难以泛化新增添的数据；有的时候模型采用的学习算法并不适用于特定的数据结构；有的时候样本本身可能有太多噪点或样本过少，使得模型无法准确地预测目标变量。这些情况下我们会说模型是欠拟合的。
